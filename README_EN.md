# AIHub2API: AI Account Management Platform / AI API Gateway

[中文](./README.md) | English

Official website: [https://aihub2api.com](https://aihub2api.com)

AIHub2API is an **AI account management platform** and **AI API gateway** for individuals and small teams. Its core value is simple: **you can import and manage your own OpenAI, Claude, Gemini and Codex accounts without self-hosting a proxy server, API relay, shared IP system or complex server stack.**

If you are searching for any of these, AIHub2API is built for that use case:

- AI account management platform
- AI API gateway

- Codex proxy
- shared proxy
- shared IP
- import your own OpenAI account
- import your own Claude account
- import your own Gemini account
- import your own Codex account
- unified API key management
- AI usage tracking and billing dashboard

## What Is AIHub2API

Many users want to manage multiple AI subscriptions and accounts in one place, but the usual path is painful: build a proxy server, configure Docker, database, Redis, reverse proxy, routing, monitoring, maintenance and networking.

AIHub2API removes that barrier. Instead of asking users to self-host an AI relay system, it provides an already-built account management and API gateway environment where users can import their own accounts and manage them directly.

## Core Features

- Import and manage your own OpenAI, Claude, Gemini and Codex accounts
- Use pre-configured shared proxy and shared IP capabilities
- No need to self-host a proxy server or API relay stack
- Unified API key generation and management
- Account status, usage, billing and request visibility
- Account management, IP management and shared proxy binding management
- Subscription, renewal and expansion workflows

## Who It Is For

- Users looking for an AI account management platform instead of building infrastructure from scratch
- People who want to manage multiple OpenAI, Claude, Gemini or Codex accounts in one place
- Users who do not want to maintain servers, proxies, shared IPs and reverse proxy systems
- Individuals and small teams that want to keep control of their own AI accounts
- Users who need an AI API gateway with unified API keys and usage tracking

## FAQ

### Do I need to self-host a proxy server?

No. That is one of the main advantages of AIHub2API. The relay and management layer is already built, so ordinary users do not need to deploy and maintain their own proxy server.

### Can I import my own AI accounts?

Yes. AIHub2API is designed around that idea. You can import and manage your own OpenAI, Claude, Gemini and Codex accounts instead of relying only on public platform accounts.

### What can I manage on the platform?

You can manage AI accounts, proxies, shared IP bindings, API keys, usage records, billing information and related permissions.

### What platforms are relevant to AIHub2API?

The current positioning focuses on OpenAI, Claude, Gemini and Codex account management and unified API access.
